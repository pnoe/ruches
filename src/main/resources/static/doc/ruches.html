1Connexion

Initialement, quand on clique sur un des items des menus le formulaire de connexion apparaît.

Si vous avez oublié votre mot de passe, cliquez sur "Mot de passe oublié ?", renseignez l’e-mail que vous avez déclaré pour ce compte

 puis cliquez sur "Réinitialisation du mot de passe". Un e-mail vous sera envoyé avec un lien à cliquer pour ré-initialiser votre mot de passe.

Une fois connecté si vous voulez modifier le nom, le mot de passe ou l’e-mail, cliquer sur le menu Personne.

2Ruches

Les ruches doivent être identifiées par un nom. Un nom court est souhaitable : c’est plus facile à écrire sur la ruche et ça prend moins de place dans les tableaux du programme.

Une ruche peut contenir un essaim et elle est placée dans un rucher (au dépôt si elle ne contient pas d’essaim). Des hausses peuvent être placées sur la ruche.

2.1Liste

La liste peut être exportée au format CSV (tableur) ou PDF, imprimée, triée, filtrée, paginée. Les colonnes peuvent être masquées/affichées.

La liste des ruches affiche les champs suivants :

  • •Nom : nom de la ruche (lien vers la page détail de la ruche) 

  • •Essaim : nom de l’essaim (lien vers la page détail de l’essaim) 

  • •Rucher : nom du rucher (lien vers la page détail du rucher) 

  • •Date : date d’ajout de cette ruche dans le rucher 

  • •Acquisition : date d’acquisition de la ruche 

  • •Poids : poids de la ruche vide 

  • •Cadres : nombre de cadres (si à jour dans les événements cadre de la ruche) 

  • •Cadres max : nom de cadres max du type de ruche 

  • •Type : type de ruche 

  • •Hausses : nombre de hausses 

  • •Commentaire ruche 

2.2Création

Vous pouvez créer une nouvelle ruche en cliquant sur le "+" dans la liste des ruches.

Le formulaire de création présente les champs suivants :

  • •Nom : le nom de la ruche. Ce champ doit être renseigné. 

  • •Active : en création on met normalement la ruche active. En modification, plutôt que de supprimer une ruche, on l’inactive. De cette manière elle n’apparaît plus dans les listes, mais elle existe toujours comme référence pour l’historique d’un rucher ou pour les récoltes passées. 

  • •Poids : le poids de la ruche vide en kg. Ce champ doit être renseigné. Peser la ruche avec tous ses cadres et un couvercle. Ce poids pourra alors être utilisé pour estimer les réserves de l’essaim pour l’hivernage après une pesée (voir essaim, événement pesée). 

  • •Type : le type de la ruche, voir menu "Types de ruche". Par exemple Dadant 10 cadres. 

  • •Date d’acquisition : la date d’achat, d’acquisition de la ruche. 

  • •Latitude : la latitude de la ruche (nombre décimal entre -90 et 90 degrés). Vous pouvez laissez ce champ tel quel ainsi que la longitude et déplacer la ruche par la suite sur la carte. 

  • •Longitude : longitude de la ruche (nombre décimal entre -180 et 180 degrés). 

  • •Commentaire : un commentaire sur la ruche. Vous pouvez par la suite créer de événements commentaire ruche datés indépendamment de ce champ. Ne pas mettre de commentaire sur l’essaim. 

Pour abandonner sans créer la ruche, cliquer sur un item des menus ou sur la touche pour revenir à la page précédente de votre navigateur.

Après avoir cliqué sur le bouton "Enregistrer", la ruche est créée et la page de détail de cette ruche affichée.

La ruche est mise dans le rucher dépôt. Un événement daté indiquant la date de mise au dépôt de cette nouvelle ruche est aussi créé.

2.3Commentaire par lot

Sélectionner dans la liste les lignes des ruches à traiter en cliquant dans ces lignes puis cliquer sur le bouton "Ajouter des événements Commentaires".

Dans le formulaire saisir :

  • •la date de l’événement 

  • •un commentaire 

Sélection des lignes :

  • •cliquer dans une zone de la ligne qui n’a pas de lien 

  • •utiliser un clic simple pour ajouter ou retirer une ligne à la sélection 

  • •utiliser "majuscule clic" pour ajouter une série de lignes qui se suivent. 

Les événements "Ruche Commentaire" sont créés pour les ruches sélectionnées.

2.4Détail d'une ruche

On accède à cette page en cliquant sur le nom d’une ruche (pages liste des ruches ou des essaims, détail d’un essaim…). Elle donne une synthèse de l’état de la ruche :

Premier tableau rappel des caractéristiques de la ruche :

Acquisition, Poids, Cadres max, Type, Latitude, Longitude, Commentaire

Deuxième tableau le rucher :

Rucher : le nom du rucher

Date : la date où la ruche a été mise dans ce rucher

Commentaire : le commentaire de cet événement mise en rucher

puis l’essaim :

Essaim : nom de l’essaim

Date : la date où l’essaim a été mis dans la ruche

Commentaire : le commentaire de cet événement mise en ruche

Troisième tableau les derniers événements :

Commentaire ruche : Commentaire, Date

Tableau des hausses de la ruche :

Nom : le nom de la hausse

Ordre : le numéro d’ordre de la hausse (1 première hausse posée sur la ruche)

Date pose, Acquisition, Poids, Cadres, Cadres max, Commentaire

 

Les boutons d’action en haut à droite de la page détail d’une ruche :

2.4.1Modifier

Les champs affichés sont les mêmes que pour la création d’une ruche.

2.4.2Cloner

Pour créer une série de ruches avec les mêmes caractéristiques que celles de la ruche courante. Il faut donner une liste de noms séparés par des virgules. Par exemple "r1,r2,r3" pour créer les ruches r1, r2 et r3.

Les nouvelles ruches sont dispersées dans le rucher. On peut les déplacer en utilisant la carte du rucher. Les événements "Ajout dans le rucher" de ces ruches sont créés.

Attention à bien partir d’une ruche avec des caractéristiques recherchées pour les clones. Les champs conservés sont :

  • •la date d’acquisition 

  • •le poids à vide 

  • •le rucher 

  • •le commentaire 

2.4.3Ajouter/Retirer/Réordonner des hausses

    • ◦permet d’ajouter ou d’enlever les hausses pour la ruche courante. Il faut cliquer sur le nom de la hausse. Un événement ajout ou retrait de hausse est créé. 

    • ◦les hausses posées peuvent aussi être ré-ordonnées par glisser-déposer. 

2.4.4Changer de rucher

Pour mettre la ruche dans un autre rucher. Le formulaire demande la date de ce mouvement, le nom du nouveau rucher et un commentaire éventuel. Un événement "Ajout dans rucher" est créé.

2.4.5Supprimer

Supprimer ne devrait être utilisé que pour annuler la création ou le clonage d’une ruche.

Si la ruche est référencée dans une récolte elle ne pourra être supprimée, vous pouvez par contre l’inactiver (modifier, cocher la case "Inactive").

En cas de suppression, les hausses sont enlevées de la ruche, les événements associés à cette ruche sont supprimés.

Attention : si un essaim était dans la ruche, il est alors hors ruche et tous ses événements associés à cette ruche sont supprimés.

2.4.6Liste des événements

Les événements permettent de mémoriser avec une date les actions effectuées sur les ruches.

La liste de tous les événements référençant cette ruche.

      • ▪Date : la date de l’événement avec heure et minute 

      • ▪Événement : le type de l’événement 

      • ▪Ruche : la ruche 

      • ▪Essaim : l’essaim (s’il y a un essaim dans la ruche) 

      • ▪Rucher : le rucher 

      • ▪Hausse : la hausse s’il s’agit d’un événement ajout ou retrait de hausse 

      • ▪Valeur : une valeur caractérisant l’événement (poids, nombre de cadres...)  

      • ▪Commentaire 

2.4.7Création d’un événement commentaire

Saisir la date et le commentaire dans le formulaire.

Le nom de la ruche concernée est rappelé en haut dans le formulaire. Vous pouvez cliquer sur ce nom pour abandonner la saisie et revenir à la page de détail de la ruche.

3Types de ruche

Les types de ruche sont utilisés pour caractériser les ruches (champ "Type").

3.1Liste

La liste des types de ruche affiche les champs suivants :

  • •Nom : le nom du type de ruche (par exemple Dadant, Langstroth, Cloake20..) 

  • •Cadre max : le nombre de cadres max de ce type de ruche 

3.2Création

Vous pouvez créer un nouveau type de ruche en cliquant sur le "+" dans la liste des types de ruches.

Le formulaire de création présente les champs suivants :

  • •Nom : ce champ doit être renseigné 

  • •Cadres max : ce champ doit être renseigné 

3.3Modifier

Les champs affichés sont les mêmes que pour la création.

4Essaims

Un essaim est composé d’une reine et des abeilles de la colonie placées dans une ruche.

La reine étant une caractéristique majeure de l’essaim (reproduction, production…), si on change de reine il faut créer un nouvel essaim dans le logiciel Ruches.

Le nom de l’essaim peut donc être vu comme le nom de la reine.

4.1Liste

La liste peut être exportée au format CSV ou PDF, imprimée, triée, filtrée, paginée. Les colonnes peuvent être masquées/affichées.

La liste des essaims affiche les champs suivants :

  • •Nom : nom de l’essaim (lien vers la page détail de l’essaim) 

  • •Ruche : nom de la ruche (lien vers la page détail de la ruche) 

  • •Rucher : nom du rucher (lien vers la page détail du rucher) 

  • •Acquisition : date d’acquisition de l’essaim 

  • •Reine marquée : oui ou non 

  • •Couleur de marquage : la couleur du marquage de la reine (même si la reine n’est pas effectivement marquée). Les couleurs reviennent de façon cyclique pour marquer les reines. Reine née les années se terminant par : 

    • ◦0 ou 5 bleu 

    • ◦1 ou 6 blanc 

    • ◦2 ou 7 jaune 

    • ◦3 ou 8 rouge 

    • ◦4 ou 9 vert 

  • •Date de naissance : la date de naissance de la reine (qui détermine la couleur de marquage à utiliser) 

  • •Souche : le nom de l’essaim souche de la reine (permet de trace le graphe de descendance) 

  • •Agressivité : de 1 à 5, 5 très agressive 

  • •Propreté : de 1 à 5, 5 très propre 

  • •Commentaire essaim 

4.2Création

Vous pouvez créer un essaim en cliquant sur le "+" dans la liste des essaims.

Le formulaire de création présente les champs suivants :

  • •Nom 

  • •Reine marquée 

  • •Date de naissance 

  • •Actif :  en création on met normalement l’essaim actif. En modification, plutôt que de supprimer une essaim, on l’inactive. De cette manière il n’apparaît plus dans les listes, mais il existe toujours comme référence pour l’historique d’un rucher ou pour les récoltes passées. 

  • •Date d'acquisition 

  • •Commentaire : un commentaire sur l’essaim. Vous pouvez par la suite créer des événements commentaire essaim datés indépendamment de ce champ. 

  • •Souche 

  • •Agressivité 

  • •Propreté 

4.3Traitement d'un lot

Sélectionner dans la liste les lignes des essaims à traiter en cliquant dans ces lignes puis cliquer sur le bouton "Ajouter des événements traitement".

Dans le formulaire saisir :

  • •la date du traitement 

  • •s’il s’agit d’événements de fin de traitement, cliquer la case à cocher "Fin de traitement" 

  • •un commentaire 

Les événements "Essaim traitement" ou "Essaim fin traitement" sont créés pour les essaims sélectionnés.

Sélection des lignes :

  • •cliquer dans une zone de la ligne qui n’a pas de lien 

  • •utiliser un clic simple pour ajouter ou retirer une ligne à la sélection 

  • •utiliser "majuscule clic" pour ajouter une série de lignes qui se suivent. 

4.4Ajout de sucre par lot

Sélectionner dans la liste les lignes des essaims pour lesquels il faut ajouter une même quantité de sucre en cliquant dans ces lignes puis cliquer sur le bouton "Ajouter des événements sucre".

Dans le formulaire saisir :

  • •la date du traitement 

  • •le poids de sucre en kg 

  • •un commentaire 

Les événements "Essaim sucre" sont créés pour les essaims sélectionnés.

4.5Commentaire par lot

Sélectionner dans la liste les lignes des ruches à traiter en cliquant dans ces lignes puis cliquer sur le bouton "Ajouter des événements Commentaires".

Dans le formulaire saisir :

  • •la date de l’événement 

  • •un commentaire 

Les événements "Essaim commentaire" sont créés pour les essaims sélectionnés.

4.6Statistiques production

Dans la liste des essaims cliquer sur le bouton "Statistiques production".

La page affiche un tableau de statistiques de production des essaims.

Les lignes affichent :

  • •Essaim : le nom de l’essaim 

  • •Acquisition : la date d’acquisition de l’essaim 

  • •Dispersion : la date de dispersion de l’essaim s’il a été dispersé 

  • •Durée : le durée de production de l’essaim en jours 

    • ◦si l’essaim a été dispersé : date acquisition – date dispersion 

    • ◦sinon : date acquisition – date du jour 

  • •Total : poids total de miel en kg sur la durée d’existence 

  • •Max : récolte max en kg 

  • •Min : récolte min en kg 

  • •Moyen : moyenne par année d’existence 

On peut filtrer les résultats :

  • •pour n’afficher que la production des essaims dans un rucher donné (on n’affiche plus la moyenne puisqu’elle dépend de la présence ou non de l’essaim dans ce rucher) 

  • •pour n’afficher que la production des essaims actifs 

4.7Statistiques âges de reines

Dans la liste des essaims cliquer sur le bouton "Statistiques âges des reines".

La page affiche un graphique à barres verticales qui donne le nombre de reines par tranches d’âge : 0-6mois, 6-12mois, 12-18mois, 24-30mois, 30-36mois, 36-42 mois, 42-48 mois, 48-54mois.

Les reines prises en compte sont celles qui sont dans des essaims actifs avec ces essaims dans des ruches.

4.8Détail d'un essaim

On accède à cette page en cliquant sur le nom d’un essaim (pages liste des ruches ou des essaims, détail d’une ruche…). Elle donne une synthèse de l’état de l’essaim :

Premier tableau rappel des caractéristiques de l’essaim :

Acquisition, Reine marquée, Couleur de marquage, Date de naissance, Souche, Agressivité, Propreté, Commentaire

Deuxième tableau, la ruche :

  • •Ruche : le nom de la ruche 

  • •Date : la date d’ajout de l’essaim dans la ruche 

  • •Commentaire : le commentaire de cet événement ajout          

Puis le rucher :

  • •Rucher : le nom du rucher 

  • •Date :  la date d’ajout de la ruche dans le rucher 

  • •Commentaire : le commentaire de cet événement ajout 

Troisième tableau les derniers événements traitement, sucre et commentaire :

  • •Traitement varroa : Début/Fin, Date, Commentaire 

  • •Ajout/Retrait de sucre : Poids, Date,        Commentaire          

  • •Commentaire : Commentaire, Date 

Troisième tableau les derniers événements pesée et cadre :

  • •Pesée : Date, Poids en kg, Commentaire 

  • •Cadres : Date, Cadres (leur nombre), Commentaire 

 

Les boutons d’action en haut à droite de la page détail d’un essaim :

4.8.1Modifier

Les champs affichés sont les mêmes que pour la création.

4.8.2Cloner

Pour créer une série d’essaims avec les mêmes caractéristiques que celles de l’essaim courant. Il faut donner une liste de noms séparés par des virgules. On peut faire suivre le nom d’un essaim par le nom de la ruche dans laquelle placer cet essaim séparé par un point virgule.

Les noms des essaims doivent ne pas être déjà utilisés.

Les ruches doivent exister et être vide d’essaim.

Par exemple la saisie de "e1;r1,e2;r2,e3" permettra de :

  • •créer l’essaim e1 et le mettre dans la ruche vide r1  

  • •créer l’essaim e2 et le mettre dans la ruche vide r2 

  • •créer l’essaim e3 

Les événements correspondants "essaim mis dans ruche" avec un commentaire rappelant le nom de l’essaim cloné sont créés.

Attention à bien partir d’un essaim avec des caractéristiques recherchées pour les clones. Les champs conservés sont :

  • •la date d’acquisition 

  • •la date de naissance de la reine 

  • •le marquage de la reine (oui/non) 

  • •l’essaim souche 

  • •l’agressivité 

  • •la propreté 

  • •le commentaire 

4.8.3Essaimer

L’essaimage demande la création d’un nouvel essaim pour la ruche concernée car la reine a changé.

Essaimer sur un essaim E d’une ruche R :

  • •Crée un nouvel essaim E_e (le suffixe _e peut être modifié à la configuration de l’application ou manuellement dans le formulaire). 

    • ◦la date d’acquisition et de naissance de la reine sont la date du formulaire 

    • ◦la souche est l’essaim E 

    • ◦la reine n’est pas marquée 

    • ◦l’agressivité et la propreté de E sont conservées. 

  • •Disperse l’essaim E (essaimage) 

  • •Met le nouvel essaim E_e dans la ruche R. 

4.8.4Associer avec une ruche

Met l’essaim courant dans une ruche. Il faut choisir la nouvelle ruche dans la liste des ruches qui est affichée. Une case à cocher permet d’afficher la liste des ruches vides d’essaim ou des ruches avec essaim. Si on choisit une ruche avec essaim, l’essaim de la ruche sera dispersé après validation avec le bouton Enregistrer du formulaire.

Puis dans le formulaire :

  • •saisir la date 

  • •un commentaire 

  • •et cocher la case "Échanger les positions des ruches"  si besoin. 

4.8.5Supprimer

Supprimer ne devrait être utilisé que pour annuler la création ou le clonage d’un essaim.

Si l’essaim est référencé dans une récolte il ne pourra être supprimé, vous pouvez par contre l’inactiver (modifier, cocher la case "Inactive").

En cas de suppression, l’essaim est enlevé de sa ruche et les événements associés à cet essaim sont supprimés.

4.8.6Graphe de descendance

Le graphe de descendance complet de cet essaim (grâce au champ Souche des essaims). Les noms des essaims sont affichés sur les nœuds du graphe. En mettant le pointeur des la souris sur un nœud, une fenêtre s’ouvre affichant :

  • •le nom de l’essaim, son état actif ou inactif, sa date d’acquisition 

  • •sa ruche 

  • •son rucher 

  • •le poids de miel total en kg qu’il a produit 

  • •le poids de miel total en kg produit par lui-même et sa descendance 

4.8.7Disperser/Remérer

Pour disperser un essaim. Les champs du formulaire :

  • •Retour de la ruche au dépôt : la ruche est mise dans le rucher dépôt 

  • •Création événement 0 cadre : un événement 0 cadre sera créé pour cette ruche 

  • •Date : la date de l’événement 

  • •Commentaire 

Après validation, l’essaim est rendu "inactif" et n’est plus visible dans la liste des essaims.

Pour afficher les "inactifs" : menu Admin/Préférences, cochez "Afficher les inactifs"

Pour remérer un essaim :

On considère pour le logiciel Ruches qu’il s’agit d’un nouvel essaim. Il faut donc

  • •créer un nouvel essaim.  

  • •puis disperser l’essaim à remérer et dans le formulaire de dispersion  

    • ◦désélectionner la case "Retour de la ruche au dépôt" 

    • ◦indiquer le nouvel essaim dans le champ "Remérage". Ce champ n’apparaît que s’il existe des essaims qui ne sont pas dans des ruches.  

L’ancien essaim est dispersé et le nouvel essaim le remplace dans la ruche. Les événements dispersion et ajout du nouvel essaim dans la ruche sont créés.

Événements

Les événements permettent de mémoriser avec une date les actions effectuées sur les essaims.

4.8.8Liste des événements

La liste de tous les événements référençant cet essaim.

Le formulaire rappel le nombre de cadre et la date du dernier événement Cadre. Il faut saisir la date, le nombre de cadres et un commentaire. Dans le commentaire vous pouvez détailler l’état des cadres en utilisant les lettres : B bâti, G gaufré, A amorcé, M Miel, P Pollen, I indéterminé, V vide, T partition, Couvain operculé O, ouvert U

7Récoltes

Une récolte consiste en l’extraction du miel d’une série de hausses.

Les hausses doivent être dans leur situation de production dans le logiciel, c’est à dire sur la ruche de production et la ruche dans le rucher de production. On ajoute alors ces hausses à la récolte ce qui permet de mémoriser dans des hausses de récolte l’essaim et le rucher de production. Les hausses doivent être pesées avant et après extraction du miel pour calculer le poids de miel extrait.

Après leur affectation à la récolte, les hausses peuvent être enlevées de leurs ruches et les ruches changées de rucher si elles ont été retirées du rucher de production.

Ce suivi de la production permet de connaître la production de miel des essaims et des ruchers.

7.1Liste

La liste peut être exportée au format CSV ou PDF, imprimée, triée, filtrée, paginée. Les colonnes peuvent être masquées/affichées.

La liste des récoltes affiche les champs suivants :

  • •Date : la date de la récolte 

  • •Rucher : le ou les ruchers dans lesquels les hausses récoltées se trouvaient 

  • •Type de miel : le type de miel (Toute fleurs, Lavande, Garrigue...)  

  • •Poids de miel mis en pot : en kg 

  • •Commentaire 

7.2Création

Vous pouvez créer une nouvelle récolte en cliquant sur le "+" dans la liste des récoltes.

Le formulaire de création présente les champs suivants :

  • •Date 

  • •Type de miel 

  • •Poids de miel : le poids de miel mis en pot. Normalement inconnu au moment de la création. Laisse ce champ vide. 

  • •Commentaire 

7.3Statistiques production essaim/récolte

Affiche un tableau donnant les productions (poids de miel en kg) de chaque essaim pour chaque récolte. Le total par essaim et le total par récolte sont affichés à droite et en bas du tableau.

7.3.1Statistiques production annuelle

Diagramme à barres verticales (histogramme) donnant pour chaque année le poids de miel en kg :

  • •Hausses, poids mesuré en pesant les hausses avant et après récolte. 

  • •Pots, poids de miel mis en pot. 

  • •Hausses/Essaim, poids de miel par hausse rapporté au nombre d’essaim actifs de l’année. Les valeurs sont sur l’axe vertical à droite. 

En positionnant le curseur de la souris sur une barre on affiche les poids exacts.

En cliquant sur un des items de la légende, on masque les barres correspondantes.

7.4Détail d'une récolte

On accède à cette page en cliquant sur le nom d’une récolte dans la liste des récoltes. Elle donne une synthèse de la récolte :

Tableau des caractéristiques de la récolte :

  • •Type de miel 

  • •Rucher 

  • •Poids de miel hausses : la somme des poids de miel résultants du pesage des hausses 

  • •Poids de miel mis en pot 

  • •Commentaire 

  • •Ruches : le nombre de ruches concernées par cette récolte 

  • •Poids de miel moyen par ruche : en kg 

Tableau des hausses de la récolte :

  • •Hausse : le nom de la hausse récoltée 

  • •Ruche : le nom de la ruche sur laquelle était cette hausse avant récolte 

  • •Essaim : le nom de l’essaim 

  • •Rucher : le nom du rucher 

  • •Poids de miel : le poids de miel en kg récolté dans cette hausse 

  • •Poids avant récolte : poids de la hausse en kg avant récolte 

  • •Poids après récolte : poids de la hausse en kg après récolte 

 

Les boutons d’action en haut à droite de la page détail d’une récolte :

7.4.1Modifier

Date, Type de miel, Poids de miel, Commentaire

7.4.2Choix des hausses

Pour ajouter ou retirer des hausses à la récolte, cliquer dans les lignes pour les sélectionner puis cliquer sur le lien "Ajouter des hausses" ou "Retirer des hausses. Sélection des lignes :

  • •cliquer dans la ligne pour ajouter ou retirer une ligne à la sélection 

  • •utiliser "majuscule clic" pour ajouter une série de lignes qui se suivent. 

Ces actions n’ont de sens qu’au moment de la récolte. Les hausses doivent être dans leur état avant récolte, c’est à dire sur la ruche et dans le rucher de production. Ainsi les valeurs mémorisées : essaim et rucher permettront de calculer des statistiques de production.

7.4.3Retrait des hausses

Cette action retire tous les hausses de la récolte de leurs ruches.

La date de retrait des hausses est demandée. Cette date sera affectée aux événements retrait des hausses.

Des événements 0 cadre sont aussi générés. La date de ces événements cadre est la date courante ou la date décalée si on l’a activée.

Les commentaires de ces événements indiquent la date de la récolte et le poids de miel récolté dans la hausse s’il a été saisi avant le retrait.

Si on traite plusieurs lot de hausses retirées de leurs ruches à des dates différentes et que l’on veut avoir des dates de retrait précises, on peut :

  • •ajouter à la récolte un lot 

  • •mesurer le poids de miel de ces hausses (facultatif mais mieux pour les commentaires des événements) 

  • •retirer toutes les hausses de ce lot de leurs ruches. 

  • •Puis passer au lot suivant. 

Il ne faut pas utiliser cette fonctionnalité après que les hausses aient été réaffectées à d’autres ruches. Pour éviter des erreurs, son accès est désactivé 30 jours après la récolte.

7.4.4Statistiques essaim

Affiche un diagramme en camembert indiquant la part de chaque essaim dans la production de miel de la récolte. En positionnant le curseur de la souris sur un secteur du camembert on affiche le nom de l’essaim, le poids de miel produit et le pourcentage que représente ce poids par rapport à la récolte.

En cliquant sur un des items de la légende on enlève le secteur correspondant du camembert.

7.4.5Supprimer

Supprime complètement la récolte et ses hausses de production.

8Événements

Les événements permettent de mémoriser les actions ou les remarques sur les ruches, ruchers, hausses et essaims.

Les types d'événements l’on peut créer par les menus des ruches, essaims, hausses et ruchers sont

  • •pour les ruches : commentaire 

  • •pour les essaims : commentaire, ajout de sucre, traitement (varroa), pesée et modification des cadres  

  • •pour les hausses : commentaire et remplissage 

  • •pour les ruchers : commentaire 

D'autres événements sont créés lorsque l'on effectue certaines actions :

  • •Ajout d'une ruche dans un rucher 

  • •Essaim mis dans une ruche 

  • •Ajout de hausse sur une ruche 

  • •Retrait d'une hausse d'une ruche 

  • •Dispersion d'un essaim 

8.1Liste

Des listes spécifiques d’événements sont disponibles dans les différents menus.

Dans la page de détail d’un essaim par exemple, la liste des événements listera uniquement les événements de l’essaim.

Le menu principal Événements affiche des sous-menus pour lister tous les événements ou uniquement les événements d’un certain type.

Un filtre permet de n’afficher que les événements pour une certaine période de temps dans ces listes (moins d’un an, d’un mois, d’une semaine…).

La liste de tous les événements affiche les champs suivants :

  • •Date : la date de l’événement 

  • •Événement : le type de l’événement 

  • •Ruche 

  • •Essaim 

  • •Rucher 

  • •Hausse 

  • •Valeur 

  • •Commentaire 

Les listes spécifiques à un type d’événement disponibles dans le menu Événements sont :

  • •Sucre : affiche les événements sucre. Le poids total et par page est affiché. Ce poids prends en compte les filtres appliqués. 

  • •Traitement varroa : affiche les événements début de traitement 

  • •Poids ruche : affiche le poids mesuré en kg, le poids de la ruche vide et la différence (permettant d’estimer les réserves pour l’hiver) 

  • •Cadre ruche 

  • •Remplissage hausse 

  • •Ajout d’une ruche dans un rucher 

8.2Création

Des créations spécifiques d’événements sont disponibles dans les différents menus.

Dans la page de détail d’un essaim par exemple, on aura accès à la création d’un événement "Traitement" ou "Sucre" pour l’essaim concerné.

À partir de la liste de tous les événements, on peut créer un événement librement en choisissant tous les champs d’un événement (mêmes champs que pour la liste). Attention, il faut normalement utiliser les créations d’événements proposées dans les pages de détail des entités pour créer les événements. Ce faisant, les objets ruche, rucher, essaim et hausse sont automatiquement renseignés.

Les champs affichés sont les mêmes que pour la création
